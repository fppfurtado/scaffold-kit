# Pitches

Aqui ficam as **propostas de ciclos** de desenvolvimento (em geral **4–6 semanas** — ajuste ao ritmo da dupla ou solo).

Cada pitch traduz ideias já um pouco validadas (discovery, spikes) em **entregas e riscos visíveis** para o próximo ciclo, sem exigir planejamento excessivo no dia zero do projeto.

## Rastreio (linha de contexto)

Preencha **Upstream** no pitch com links para discovery e/ou spikes que o motivam; quando o ciclo existir, **atualize a lista downstream** no discovery de origem (mesmo padrão dos spikes). Objetivo: grafo legível *upstream → downstream* só com links em Markdown, sem ficheiro índice obrigatório.

## Como criar um pitch

- Use `cycles/cycle-1/PXXX-pitch-template.md` como modelo (veja `cycles/cycle-1/README.md` para `NXXX` e `RXXX` e a convenção de nomes)
- Defina objetivo, escopo, riscos e estimativa realista para o tamanho do time
- Após “aprovação” (mesmo que seja só você), trabalhe o ciclo na pasta `cycles/cycle-1/` com notas e review a partir de `NXXX-note-template.md` e `RXXX-review-template.md` — ou copie `cycle-1/` para `cycle-2/` no próximo ciclo
