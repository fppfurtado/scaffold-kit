# Ciclo 1 — pasta de trabalho

Este diretório é o **primeiro ciclo** sugerido (em geral **4–6 semanas** no ritmo de solo ou dupla). A ideia é manter **poucos arquivos fixos** e muita clareza sobre o que entra e o que fica de fora.

| Arquivo     | Quando usar |
| ----------- | ----------- |
| `pitch.md`  | Antes e durante o ciclo: problema, escopo delimitado, riscos e critérios de pronto. |
| `notes.md`  | Durante o ciclo: decisões rápidas, cortes de escopo, bloqueios — sem minutar reunião. |
| `review.md` | Ao fechar: o que entregou, aprendizados, próximo passo (próximo ciclo, spike ou pivot). |

**Filosofia do template:** não transforme isto em gestão pesada. Se uma seção do `pitch.md` não ajuda, apague. Links úteis: `docs/strategy/discovery/`, `docs/strategy/spikes/`, `docs/architecture/adrs/`.

**Próximos ciclos:** copie `cycle-1/` para `cycle-2/` (ou renomeie) e ajuste datas e título — evite um único documento “eterno” que mistura ciclos.

Quem for ajudar depois (outra pessoa ou **agente de IA**) deve conseguir entender o ciclo lendo **pitch → notes → review** em ordem.
