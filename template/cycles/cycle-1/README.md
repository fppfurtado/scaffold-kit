# Ciclo 1 — pasta de trabalho

Este diretório é o **primeiro ciclo** sugerido (em geral **4–6 semanas** no ritmo de solo ou dupla). Os modelos seguem a mesma convenção de `docs/strategy/discovery/`: **letra do tipo + `XXX` + nome descritivo** (`XXX` é placeholder; ao copiar para um ciclo real, substitua por um id, ex.: `P01`, `S02`).

| Modelo | Quando usar |
| ------ | ----------- |
| `PXXX-pitch-template.md` | Antes e durante o ciclo: problema, escopo delimitado, riscos e critérios de pronto. |
| `NXXX-note-template.md` | Durante o ciclo: decisões rápidas, cortes de escopo, bloqueios — sem minutar reunião. |
| `RXXX-review-template.md` | Ao fechar: o que entregou, aprendizados, próximo passo (próximo ciclo, spike ou pivot). |

**Como usar:** copie cada modelo, renomeie trocando `XXX` pelo número do artefato (ex.: `P01-pitch.md`, `N01-notes.md`, `R01-review.md`) e preencha. Mantenha a ordem lógica **pitch → notas → review**.

**Filosofia do template:** não transforme isto em gestão pesada. Se uma seção do pitch não ajuda, apague. Links úteis: `docs/strategy/discovery/`, `docs/strategy/spikes/`, `docs/architecture/adrs/`.

**Próximos ciclos:** copie `cycle-1/` para `cycle-2/` (ou renomeie) e ajuste datas e título — evite um único documento “eterno” que mistura ciclos.

Quem for ajudar depois (outra pessoa ou **agente de IA**) deve conseguir entender o ciclo lendo **P → N → R** na ordem acima.
